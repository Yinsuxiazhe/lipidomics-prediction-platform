# 2026-04-18 网站展示逻辑清单

## 一、当前版本（本轮 P0）必须满足

### 1. indicator 命名
- [x] 前台 indicator 主名称统一显示为变化量写法：`ΔBMI`、`ΔPBF`、`ΔPSM`、`ΔWHR` 等
- [x] 不再把 `q3 / q4 / Q / T` 混进 indicator 主名称
- [x] 不再在同一列混排“全称 + 缩写 + 后缀 + 旧内部命名”

### 2. grouping 展示
- [x] grouping 单独展示，不与 indicator 主列合并
- [x] 统一前台写法：
  - `Q（Q1 vs Q4）`
  - `T（T1 vs T3）`
- [x] 不直接向用户暴露旧内部命名 `q3 / q4`

### 3. High / Low Response 解释
- [x] 对负向更优指标（BMI / PBF / WHR / 体重 / 腰围 / 臀围 / BMI z-score）：
  - High Response = 下降更多
  - Low Response = 下降较少，甚至可能升高
- [x] 对正向更优指标（PSM）：
  - High Response = 上升更多
  - Low Response = 上升较少，甚至可能下降

### 4. 页面说明层
- [x] help / glossary / performance summary 明确写出：`Δ = outroll - enroll = 干预后 - 干预前`
- [x] 明确写出：当前上线网站仍为 **lipid-only**
- [x] 明确写出：Q / T 是 extreme-group grouping，而不是单个分位点编号

## 二、当前版本不做但必须保留在方案中的内容
- [ ] clinical-only 模型接入
- [ ] fusion 模型接入
- [ ] 按模型类型动态显示临床输入区
- [ ] clinical-only / lipid-only / fusion 三层模型对比模块
- [ ] 校准曲线、DCA、算法比较表等高级展示模块

## 三、fusion 默认展示规则（下一轮）
- [ ] 每个 indicator × grouping 下先离线筛出 best fusion model
- [ ] 优先按同一验证框架下的 AUROC 排序
- [ ] 若 AUROC 接近，再比较 AUPRC、Sensitivity、Specificity、稳健性与模型简洁度
- [ ] 若 fusion 并未明显更优，则只做客观对比，不强行写成 fusion 最优

## 四、最终一句话原则
> 当前网站先把“命名正确、分组清楚、方向解释真实”这三件事做好；clinical 与 fusion 的丰富展示放到下一轮，不提前假装已经上线。
