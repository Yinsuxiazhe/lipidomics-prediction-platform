strict nested CV 结果定位说明

本说明用于界定本包内各类结果的正式定位、使用边界与推荐表述。

1. 结果分层与定位

  1. 正式主结果：01_主图与主结果/ 中的 Figure 6-4、主结果 CSV 及其对应说明，均以 strict nested CV 为正式口径。
  2. 补充结果：02_补充分析与审计/ 中的小模型压缩结果、标签/队列审计与诊断表，用于补充说明稳定性与局限性。
  3. 关键实现依据：03_关键代码与配置/ 中的配置文件、nested CV 核心脚本和绘图脚本，构成正式结果的实现证据链。
  4. 方法学说明：本目录文档用于解释为何本包采用 strict nested CV 作为正式结果，以及其他结果应如何被引用。

2. strict nested CV 之所以作为正式口径的依据

  1. 特征筛选、标准化和参数调优均限定在训练折内完成，避免了验证信息泄漏。
  2. 外层测试折不参与模型选择，因此结果更接近模型在未知样本中的真实表现。
  3. 绘图脚本直接读取 strict nested CV 输出，而不是在绘图阶段重新建模，有助于保证图形与结果表的一致性。
  4. 对外汇报、论文主图和正式结论应优先依赖可信度更高的评估口径，而不是更高但边界不够严格的分数。

3. 开发阶段较高分结果的正式定位

  1. 先前较高分结果如需提及，只能定位为 开发阶段探索性结果。
  2. 这类结果可以用于说明“曾观察到潜在线索”或“开发过程中曾出现更高区分度”，但不宜承担正式性能声明。
  3. 在正式正文中，如必须提及，应放在 Discussion、Limitations 或方法补充部分，并明确与 strict nested CV 正式结果区分开来。
  4. 本说明包不再把此类结果作为主要入口材料，以避免正式汇报时出现结果口径混淆。

4. 正式表述时的推荐写法

4.1 推荐写法

  1. 当前 strict nested CV 结果显示，临床与脂质融合模型达到当前最高 mean AUC，但优势幅度有限。
  2. 扩展变量集没有带来更好的泛化表现，提示现阶段主要问题不是模型复杂度不足，而是信号强度与稳定性仍有限。
  3. 现阶段更合适的说法是“存在一定预测信号，但仍需进一步验证与压缩模型复杂度”。

4.2 不推荐写法

  1. 不建议直接表述为“已经建立高性能稳定预测模型”。
  2. 不建议把开发阶段较高分结果写成正式主结果或正式主图。
  3. 不建议把轻微 AUC 增量包装成明确且稳定的临床增益。

5. 本说明包的证据链对应关系

  1. 图形证据：01_主图与主结果/Figure6-4_Honest_NestedCV.png 与 PDF。
  2. 表格证据：01_主图与主结果/performance_summary.csv、fold_metrics.csv、feature_stability_summary.csv。
  3. 补充证据：02_补充分析与审计/20260310_predefined_small_model_comparison.csv 与小模型子目录。
  4. 实现证据：03_关键代码与配置/src/models/run_nested_cv.py、make_tables.py、fig6-4最终.R。
  5. 综合说明入口：00_正式说明/20260311_strict_nested_cv正式说明.html。

6. 当前阶段最建议的使用方式

  1. 将 00_正式说明/20260311_strict_nested_cv正式说明.html 作为正式对外说明首页。
  2. 将 01_主图与主结果/ 作为所有主结果图表和数据引用的唯一来源。
  3. 在需要回应方法学问题时，优先引用本目录说明，而不是引用过程性会话记录。
  4. 在需要讨论优化方向时，引用 02_补充分析与审计/ 的补充结果，但保持其“补充证据”定位不变。
