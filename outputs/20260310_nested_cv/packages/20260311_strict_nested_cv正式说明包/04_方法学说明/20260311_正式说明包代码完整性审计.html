正式说明包代码完整性审计

审计对象：03_关键代码与配置/ 目录及其与正式说明包复现能力的对应关系。

一、审计结论摘要

  1. 结论一：代码“用于说明核心方法逻辑”基本够用。
    • 当前包内已经包含主流程入口、strict nested CV 核心建模脚本、结果表导出脚本、ROC 导出脚本、主配置文件和 Figure 6-4 绘图脚本。
    • 如果目的只是让接收者理解主结果是如何生成的，这一层级的信息是基本齐全的。
  2. 结论二：代码“用于独立完整复现”并不齐全。
    • 正式包当前并不是一个可脱离原项目目录独立运行的完整复现包。
    • 也就是说，它更适合作为“方法学证据与关键实现摘录”，而不是“拿到包就能从头跑完整流程”的交付件。
  3. 结论三：当前最准确的定位应为“关键代码摘录包”，而不是“完整复现代码包”。

二、已包含的关键代码文件

  1. run_pipeline.py
    • 主流程入口。
  2. config/analysis.yaml
    • 输入路径、标签定义、CV 参数和输出目录配置。
  3. src/models/run_nested_cv.py
    • strict nested CV 的核心逻辑。
  4. src/reports/make_tables.py
    • 主结果表导出逻辑。
  5. src/reports/make_figures.py
    • ROC 点位导出逻辑。
  6. fig6-4最终.R
    • 主图 Figure 6-4 绘图脚本。

三、阻碍独立复现的关键缺口

P1：缺少多个被直接 import 的本地模块

以下模块被当前包内代码直接调用，但并未一同打包：

  1. src.data.build_cohort
  2. src.io.load_data
  3. src.features.correlation_prune
  4. src.features.univariate_screen
  5. src.preprocess.clinical_filter
  6. src.preprocess.lipid_transform

这意味着： - run_pipeline.py 在当前正式说明包中不能单独运行； - run_nested_cv.py 也不能在当前包内独立导入执行。

P1：配置文件使用原项目绝对路径，缺乏可移植性

config/analysis.yaml 中的数据路径和输出路径全部指向原项目机器上的绝对路径。

这意味着： 1. 若把正式说明包拷贝到别的电脑或别的目录，配置将立即失效； 2. 即使补齐部分代码模块，也仍然不能直接复现； 3. 当前配置更像“原始工程的运行快照”，不是“可移植的交付配置”。

P1：R 主图脚本在正式包内不能独立重跑主流程

实际测试表明： 1. 直接运行包内 run_pipeline.py --dry-run 会报 ModuleNotFoundError: No module named 'src.data'； 2. 直接运行包内 fig6-4最终.R 也会因为其内部调用 run_pipeline.py 而失败。

因此，当前包内的 fig6-4最终.R 更适合作为方法说明脚本，而不是当前包环境下可直接复现的可执行脚本。

四、次级缺口

P2：缺少环境依赖说明

当前正式包中没有以下文件：

  1. requirements.txt / environment.yml / pyproject.toml
  2. R 包依赖说明文件
  3. Python 与 R 版本说明

这会导致接收者即使补齐代码，也不容易准确还原运行环境。

P2：缺少测试文件

正式包没有打入对应测试文件，因此： 1. 无法在包内直接验证代码行为是否一致； 2. 无法把“代码可看”提升为“代码可验证”。

P2：当前代码目录中原本夹带 .DS_Store

这属于低风险问题，但会影响交付整洁度。当前本轮整理时已清理。

五、当前包适合承担的角色

适合

  1. 解释 strict nested CV 的核心方法流程；
  2. 说明主图和结果表来自哪些关键脚本；
  3. 作为正式说明包中的“关键实现证据”。

不适合

  1. 作为完全独立的复现代码仓；
  2. 作为别人拿到后即可一键重跑的完整分析包；
  3. 作为软件交付意义上的完整 source release。

六、建议的正式表述

建议把当前 03_关键代码与配置/ 的定位写成：

本目录提供 strict nested CV 主结果所对应的关键实现脚本与配置文件，用于说明主结果生成逻辑与方法学边界；其目的在于支撑结果解释与审阅，而非作为完整、可脱离原工程环境独立运行的复现包。

七、如果要升级成“完整复现包”，至少还需要补什么

  1. 补齐所有 src.data、src.io、src.features、src.preprocess 相关模块；
  2. 增加环境依赖文件；
  3. 把绝对路径改成相对路径或参数化路径；
  4. 补入最小测试集与运行说明；
  5. 增加一个真正可在包内运行的 README 与复现命令示例。
