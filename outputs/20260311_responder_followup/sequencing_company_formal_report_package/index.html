一句话结论

当前正式主结果应以 strict nested CV 的 outer-test performance 为准；如果此前沟通中提到过 AUC≈0.8，那对应的是同一 strict nested CV 框架中的 mean_train_auc，不是 outer-test 或 external validation AUC。

主结果怎么说最稳妥

  • 当前数据下，严格 nested CV 给出的正式主结果约为 AUC 0.50–0.54。
  • 这说明模型在当前样本与当前标签定义下，已经能观察到一定建模信号，但 正式泛化性能仍属中等，不应直接表述为稳定的高性能分类器。
  • 因此，对外正式沟通时，建议始终以 outer-test mean AUC 作为主口径。
推荐表述：“正式主结果基于 strict nested CV 的 outer-test performance；目前主模型 outer-test mean AUC 约为 0.50–0.54。”

为什么又会看到 AUC≈0.8

这一点最容易引起误解，所以建议单独解释。

模型 outer-test mean AUC mean_train_auc 应如何理解
fusion_elastic_net 0.5364 0.8055 训练折内可学到较强模式，但外层测试泛化明显下降。
fusion_full_elastic_net 0.5258 0.8496 0.8+ 数值来自训练阶段平均区分度，不是正式外层验证结果。
对外时建议直接说明：“AUC≈0.8 对应 strict nested CV 中的 mean_train_auc；正式性能声明仍以 outer-test mean AUC 为准。”

当前结论边界

  • 本包中的 repeated hold-out、leave-one-prefix-out 与 post-intervention within-dataset temporal validation，均作为补充稳健性证据使用。
  • 这些分析有助于理解模型稳定性、标签一致性与时相可迁移性，但 不等同于 external validation。
  • 因此，本轮交付更适合作为正式技术沟通与后续方案讨论基础，而不是对外宣称“已完成独立外部验证”的材料。
