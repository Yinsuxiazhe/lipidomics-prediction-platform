# Responder/Non-responder 新分析方案 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 基于 2026-03-11 对接中新形成的共识，重构后续脂质组学预测分析路线：优先维护“分组可自圆其说且基线尽量平衡”的主口径，同时加入出组/随机抽样自我验证、小模型稳定性优化和机制表型扩展。

**Architecture:** 本轮不再把“继续堆模型、追求更高训练 AUC”作为主线，而是采用“分组口径审查 → 验证设计增强 → 小模型稳健化 → 机制补充”的四段式路线。主分析继续基于基线数据预测 responder / non-responder，但新增两类验证思路：一是用出组数据评估人群异质性是否稳定存在，二是用随机抽样/留出法做自我验证；若缺少真正外部队列，则以 pseudo-external 和 targeted replication 作为替代路径。

**Tech Stack:** Python strict nested CV pipeline、现有 CSV 队列文件、Markdown/HTML 说明文档、补充统计图（瀑布图/直方图/差异分析表）。

---

## 一、总原则（从对接内容提炼）

1. **主口径暂不轻易推翻当前 responder 分组。**
   - 当前分组是“多轮尝试后，能够自圆其说且基线无显著差异”的结果。
   - 因此不能只为了拉大组间差异而贸然换分组，否则会引入“基线不平”的审稿风险。

2. **模型优化不再以“继续堆算法”作为唯一主线。**
   - 训练集 AUC 已可到 0.8+，但 test/outer-test 表现偏低，说明问题更像泛化不足而不是拟合不足。
   - 后续重点应放在：验证策略增强、标签稳定性、模型压缩、结果可解释性。

3. **外部验证不是唯一出路。**
   - 由于儿童脂质组外部队列稀缺、脂质分子不易完全匹配，允许先做更强的内部验证/自我验证。
   - 真正外部队列可放到文章雏形出来后，再对外寻求合作。

4. **机制部分准备扩展到心血管相关表型。**
   - 该部分不作为当前预测主分析的前置条件，但可作为后续机制/临床表型连接的重要补充方向。

---

## 二、新分析方案（建议作为后续主路线）

### 模块 A：当前 responder / non-responder 主分组口径固化 + 证据化说明

**目标：** 不急着改分组，先把“为什么保留当前分组”说清楚。

**核心动作：**
1. 回溯当前 responder / non-responder 分组依据。
2. 整理“当前分组经过多轮尝试后保留”的理由。
3. 补一套能支撑当前分组合理性的图表和说明：
   - ΔBMI（或当前使用的核心响应指标）瀑布图；
   - 分布直方图/密度图；
   - 基线平衡性汇总表；
   - 当前分组与备选分组的对照说明表。
4. 在方法学里明确：
   - 当前主分组优先考虑**临床可解释性 + 基线平衡性 + 可投稿防守性**；
   - 而不是单纯为了机器学习可分性最大化。

**交付物建议：**
- `outputs/.../group_definition_audit.md`
- `outputs/.../group_definition_comparison.csv`
- 瀑布图、直方图、基线平衡表

**决策门槛：**
- 如果当前主分组能维持较好基线平衡，则继续作为主分析口径；
- 如果发现原始定义存在明显灰区/不一致，再进入模块 B 的分层敏感性分析。

---

### 模块 B：备选 responder 分组的敏感性分析（不是替代主口径，而是补充）

**目标：** 回应“是否可以尝试别的 responder 分组”这个问题，但避免直接冲掉当前主分组。

**核心动作：**
1. 在当前主分组之外，设计 2–3 套备选分组策略：
   - 更严格阈值分组；
   - 排除灰区样本后的二分类；
   - 连续结局衍生的高/低响应极端组方案。
2. 对每套备选分组，统一比较：
   - 基线平衡性；
   - 样本量损失；
   - strict nested CV outer-test AUC；
   - train-test gap；
   - 特征稳定性。
3. 明确主次定位：
   - **主文仍用当前主分组**；
   - **备选分组只放补充材料/敏感性分析**。

**交付物建议：**
- `alternative_grouping_sensitivity.md`
- `alternative_grouping_performance.csv`
- `alternative_grouping_baseline_balance.csv`

**决策门槛：**
- 如果某套备选分组显著提升泛化且基线仍可接受，可作为 Discussion 中的“更严格定义下结果更稳”的补充论据；
- 如果只是提高可分性但基线明显失衡，则仅保留为 exploratory 结果，不升级为主分析。

---

### 模块 C：用“出组数据”做验证（当前对接中的新增重点）

**目标：** 利用出组/干预后数据验证：当前 responder / non-responder 差异是否反映的是较稳定的个体异质性，而非仅仅是运动过程中的偶然变化。

**说明：**
- 这里的“out 数据”我当前按对接语境理解为**出组/干预后数据**；正式执行前需要先确认对应的数据文件和字段。

**核心动作：**
1. 明确出组数据的文件、变量映射与可用样本数。
2. 设计两类验证问题：
   - **验证 1：** 用当前分组标签，看出组时这些人群是否仍维持某些稳定差异；
   - **验证 2：** 用基线模型在出组数据上做迁移/投射，观察区分模式是否仍存在。
3. 重点看：
   - 差异方向是否一致；
   - 候选脂质/临床锚点是否仍稳定；
   - 如果出组也能区分，是否支持“人群固有异质性”这一解释。

**交付物建议：**
- `outcome_phase_validation.md`
- `baseline_vs_out_validation.csv`
- 出组阶段差异图/箱线图/森林图

**解释边界：**
- 若出组数据支持差异延续，可增强“人群异质性稳定存在”的论点；
- 但这仍不等于真正外部验证，应明确写成“内部时相验证/出组验证”。

---

### 模块 D：随机抽样 / 留出法自我验证（self-validation）

**目标：** 在缺少真正外部队列时，增加一层更接近真实部署的内部验证证据。

**核心动作：**
1. 在 strict nested CV 主结果之外，再设计一套自我验证框架：
   - repeated random hold-out；
   - train/validation/test 三段式；
   - 或 Monte Carlo split。
2. 若有批次、时间、来源信息，则优先升级为 pseudo-external：
   - leave-one-batch-out；
   - leave-one-prefix-out；
   - leave-one-time-block-out。
3. 与 strict nested CV 平行汇报：
   - strict nested CV 作为正式主口径；
   - self-validation / pseudo-external 作为补充稳健性证据。

**交付物建议：**
- `self_validation_report.md`
- `pseudo_external_report.md`
- 各验证框架性能对比表

**决策门槛：**
- 若 self-validation 与 strict nested CV 方向一致，可增强结果可信度；
- 若差异很大，优先回头排查分组稳定性和数据异质性。

---

### 模块 E：模型优化只走“小模型 / 稀疏模型 / 稳定特征”路线

**目标：** 不再以扩大变量维度为主，而是优先追求更稳的泛化结果。

**核心动作：**
1. 保留当前已证明更合理的方向：
   - `ultra_sparse_lipid`
   - `compact_fusion`
   - top-k 稳定特征模型
2. 所有新模型比较统一输出：
   - mean outer-test AUC
   - std AUC
   - mean train AUC
   - train-test gap
   - selected feature count
   - selection stability
3. 不再优先做 expanded 类大模型扩展，除非是为机制解释提供特定候选变量。

**交付物建议：**
- `small_model_comparison_v2.csv`
- `feature_stability_summary_v2.csv`
- `topk_stability_sweep.csv`

**判断标准：**
- 小模型若 AUC 近似、gap 更小、稳定性更高，则优先进入论文主叙述；
- 大模型即使训练更高，只要泛化不增益，就只保留为反例/局限性说明。

---

### 模块 F：机制部分新增“心血管相关表型”桥接分析

**目标：** 响应黄老师新思路，在预测主线之外补一条“脂质变化—心血管表型”的机制/临床解释支线。

**核心动作：**
1. 等师姐整理完新增表型后，先做字段盘点：
   - 哪些是心血管相关表型；
   - 哪些是基线变量；
   - 哪些有前后变化值。
2. 明确该模块定位：
   - 是作为机制支持？
   - 还是作为额外临床锚点/解释变量？
3. 建议先做两层分析：
   - responder / non-responder 间的表型差异；
   - 稳定脂质与这些表型之间的相关/回归关联。

**交付物建议：**
- `cardiovascular_phenotype_bridge_analysis.md`
- `lipid_phenotype_association.csv`
- 机制示意图或相关热图

**边界提醒：**
- 该模块应首先服务于“解释”和“临床落脚点”，不建议一开始就并入高维预测主模型，以免再次加重过拟合。

---

### 模块 G：外部验证与合作队列放到第二阶段

**目标：** 接受“当前很难找到完全匹配外部脂质组队列”的现实，但不放弃后续合作路径。

**核心动作：**
1. 当前阶段先不把“找到外部队列”设为主线阻塞条件。
2. 文章雏形出来后，再对外展示：
   - 当前主结果；
   - 小模型稳定结果；
   - 可复测的少数脂质候选。
3. 外部合作优先级：
   - 有共享临床变量的儿童队列；
   - 能复测少数稳定脂质的小样本血样；
   - 能提供相关表型而非全量脂质组的平台。

**交付物建议：**
- `external_validation_strategy.md`
- `candidate_targeted_panel.csv`

## 三、建议的主次排布（很重要）

### 适合放主文章/主结果的
1. 当前主 responder 分组口径；
2. strict nested CV 主结果；
3. 小模型/稀疏模型中最稳的一组补充结果；
4. 出组验证若方向一致，可作为增强论点的重要补充。

### 适合放补充材料的
1. 备选 responder 分组敏感性分析；
2. 随机抽样/自我验证结果；
3. pseudo-external 结果；
4. 大模型对照；
5. 更大范围的机制探索结果。

---

## 四、推荐执行顺序（最终版）

1. **先固化当前 responder 主分组的证据链**
   - 解释为什么保留当前分组，而不是立即换。

2. **再做备选分组敏感性分析**
   - 有尝试，但不冲掉主分析。

3. **马上上“出组验证 + 随机抽样自我验证”**
   - 这是本次对接中新增加、也最务实的验证路线。

4. **模型优化只保留小模型/稀疏模型路线**
   - 不再盲目扩维。

5. **机制部分等待心血管表型整理后接入**
   - 作为机制补充支线推进。

6. **外部队列合作放到文章雏形完成后**
   - 不阻塞当前主分析推进。

## 五、可直接发给师姐的简版表述

> 根据这次新的对接思路，后续分析不建议再把重点放在继续堆模型或单纯追求更高训练 AUC，而是分成四条主线推进：第一，保留当前 responder / non-responder 主分组，并补充分组合理性与基线平衡的证据链；第二，在不替代主口径的前提下，做备选 responder 分组的敏感性分析；第三，把出组数据验证和随机抽样自我验证作为当前最务实的验证增强路线；第四，模型优化继续优先走小模型、稀疏模型和稳定特征路线。若后续心血管相关表型整理出来，再作为机制补充模块接入；真正外部队列合作则放到文章整体雏形完成后推进。
